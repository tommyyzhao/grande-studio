## Codebase Patterns
- Use `npx shadcn-svelte add <component> --no-deps --overwrite` and pipe `echo "y"` for confirmation to add shadcn-svelte components
- shadcn-svelte v1.2.7 presets: nova, vega, maia, lyra, mira, luma (not "default"/"new-york")
- `WithElementRef` type must be exported from `$lib/utils.ts` for shadcn-svelte components
- Tailwind CSS v4 uses `@tailwindcss/vite` plugin (not PostCSS) and `@theme inline` blocks for design tokens
- shadcn-svelte CSS uses oklch color format in `:root` / `.dark` selectors
- `tw-animate-css` is required for shadcn-svelte animations
- Project uses `@sveltejs/adapter-cloudflare` with `pages_build_output_dir: ".svelte-kit/cloudflare"` in wrangler.jsonc
- Drizzle ORM schema lives at `src/lib/server/db/schema.ts`; db client factory at `src/lib/server/db/index.ts`
- Local dev uses `pg` driver via `createLocalDb()`; production uses `@neondatabase/serverless` via `createNeonDb()`
- `drizzle.config.ts` reads `DATABASE_URL` from `process.env` (used by drizzle-kit CLI)
- Local Postgres database is `grande_studio` on localhost:5432
- Schema file must have at least `export {}` to be a valid TypeScript module (Drizzle requires it)
- Use `.$type<EnumType>()` on text columns for typed enums in Drizzle (keeps DB flexible, TS strict)
- Drizzle index definitions use a second arg function to `pgTable()` returning an array
- Exported type aliases (SourceType, Provider, AssetStatus, BranchType) from schema.ts are reusable across services
- Use `check()` from `drizzle-orm/pg-core` + `sql` from `drizzle-orm` for CHECK constraints in table definitions
- Custom SQL migrations (RLS, triggers, etc.): use `drizzle-kit generate --custom --name=<name>` then apply with `psql -d grande_studio -f <file>` (not `drizzle-kit push`)
- RLS helper `withRLS(db, userId, fn)` at `src/lib/server/db/rls.ts` wraps DB operations in a transaction with `SET LOCAL app.user_id`
- BetterAuth auth-schema lives at `src/lib/server/db/auth-schema.ts` â€” managed by BetterAuth, do not rename columns
- BetterAuth Drizzle adapter: use `drizzleAdapter(db, { provider: 'pg', schema: authSchema })` â€” pass the auth schema explicitly
- BetterAuth CLI `migrate` doesn't work with Drizzle adapter â€” use `npx @better-auth/cli generate` then `drizzle-kit generate` + `drizzle-kit push`
- BetterAuth reads `BETTER_AUTH_SECRET` and `BETTER_AUTH_URL` from `process.env` automatically
- Auth server at `src/lib/server/auth.ts`; auth client at `src/lib/auth-client.ts`; API route at `src/routes/api/auth/[...all]/+server.ts`
- `drizzle.config.ts` schema field accepts an array of paths for multiple schema files
- BetterAuth Svelte client API: `authClient.signUp.email({ email, password, name })` and `authClient.signIn.email({ email, password })` â€” both return `{ data, error }`
- shadcn-svelte Card uses namespace import: `import * as Card from '$lib/components/ui/card'`

- Applied migration to local Postgres â€” 8 ALTER TABLE + 32 CREATE POLICY all succeeded
- Files changed: drizzle/0003_rls_ownership_policies.sql, drizzle/meta/_journal.json, drizzle/meta/0003_snapshot.json, src/lib/server/db/rls.ts
- **Learnings for future iterations:**
  - Drizzle doesn't have first-class RLS support â€” use `drizzle-kit generate --custom --name=<name>` to create empty SQL migration files for custom DDL
  - `drizzle-kit push` only applies schema-level changes (tables, columns, indexes), NOT custom SQL migrations â€” use `psql -d grande_studio -f <migration_file>` to apply custom migrations
  - RLS policies use `current_setting('app.user_id', true)::uuid` â€” the `true` arg makes it return NULL (not error) if the setting is missing, so RLS denies all access by default (fail-closed)
  - `SET LOCAL app.user_id = '<uuid>'` scopes the setting to the current transaction â€” use `withRLS()` helper from `src/lib/server/db/rls.ts` to wrap DB operations
  - The table owner (typically the connection role) bypasses RLS by default â€” in production, use a restricted role or set `ALTER TABLE ... FORCE ROW LEVEL SECURITY` if the app connects as the table owner
