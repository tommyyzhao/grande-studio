  - Drizzle doesn't have first-class RLS support — use `drizzle-kit generate --custom --name=<name>` to create empty SQL migration files for custom DDL

  - RLS policies use `current_setting('app.user_id', true)::uuid` — the `true` arg makes it return NULL (not error) if the setting is missing, so RLS denies all access by default (fail-closed)

  - The table owner (typically the connection role) bypasses RLS by default — in production, use a restricted role or set `ALTER TABLE ... FORCE ROW LEVEL SECURITY` if the app connects as the table owner
