# shadcn-svelte Integration Guide

**Always consult [shadcn-svelte.com/docs](https://www.shadcn-svelte.com/docs) for latest API.**

> This is huntabyte's Svelte port of shadcn/ui — NOT the React version. Do NOT use `npx shadcn@latest` (React CLI). Use `npx shadcn-svelte@latest`.

---

## Setup Workflow

1. Create SvelteKit project: `npx sv create my-app --add tailwindcss`
2. Run: `npx shadcn-svelte@latest init`
3. Configure prompts: base color, CSS path, import aliases
4. Add components: `npx shadcn-svelte@latest add button`
5. Import: `import { Button } from "$lib/components/ui/button/index.js"`

---

## Quick Reference

### CLI Commands

| Command | Purpose |
|---------|---------|
| `npx shadcn-svelte@latest init` | Initialize project — installs deps, creates `cn()`, adds CSS vars |
| `npx shadcn-svelte@latest add <component>` | Add component source to project |
| `npx shadcn-svelte@latest add -a` | Add ALL components |
| `npx shadcn-svelte@latest add <component> -o` | Overwrite existing component files |

**CLI flags:** `-c, --cwd <path>` (working dir), `--no-deps` (skip install), `-y, --yes` (skip prompts), `-o, --overwrite`.

### Init Options

| Option | Notes |
|--------|-------|
| `--base-color` | slate, gray, zinc, neutral, stone |
| `--css <path>` | Path to global CSS file (default: src/app.css) |
| `--components-alias` | Default: `$lib/components` |
| `--utils-alias` | Default: `$lib/utils` |
| `--hooks-alias` | Default: `$lib/hooks` |
| `--ui-alias` | Default: `$lib/components/ui` |

### components.json

Generated by `init`. Controls where components are installed and how imports resolve:

```json
{
  "$schema": "https://shadcn-svelte.com/schema.json",
  "style": "default",
  "tailwind": {
    "css": "src/app.css",
    "baseColor": "zinc"
  },
  "aliases": {
    "components": "$lib/components",
    "utils": "$lib/utils",
    "hooks": "$lib/hooks",
    "ui": "$lib/components/ui"
  }
}
```

### The cn() Utility

Located at `$lib/utils.ts` (created by init):

```typescript
import { clsx, type ClassValue } from "clsx";
import { twMerge } from "tailwind-merge";

export function cn(...inputs: ClassValue[]) {
  return twMerge(clsx(inputs));
}
```

Use `cn()` to merge Tailwind classes without conflicts:
```svelte
<div class={cn("px-4 py-2", conditional && "bg-primary", className)}>
```

## Tailwind v4 Integration

shadcn-svelte uses **Tailwind CSS v4** with CSS-first configuration. There is NO `tailwind.config.js`.

### Vite Configuration

```typescript
// vite.config.ts
import { sveltekit } from "@sveltejs/kit/vite";
import tailwindcss from "@tailwindcss/vite";
import { defineConfig } from "vite";

export default defineConfig({
  plugins: [
    tailwindcss(),  // MUST come before sveltekit()
    sveltekit(),
  ],
});
```

### CSS Variables in app.css

Theme colours use CSS variables with oklch values. The `@theme inline` block registers them with Tailwind v4:

```css
@import "tailwindcss";

:root {
  --background: oklch(1 0 0);
  --foreground: oklch(0.145 0 0);
  --primary: oklch(0.205 0 0);
  --primary-foreground: oklch(0.985 0 0);
  --secondary: oklch(0.97 0 0);
  --secondary-foreground: oklch(0.205 0 0);
  --muted: oklch(0.97 0 0);
  --muted-foreground: oklch(0.556 0 0);
  --accent: oklch(0.97 0 0);
  --accent-foreground: oklch(0.205 0 0);
  --destructive: oklch(0.577 0.245 27.325);
  --border: oklch(0.922 0 0);
  --input: oklch(0.922 0 0);
  --ring: oklch(0.708 0 0);
  --radius: 0.625rem;
  /* chart-1 through chart-5, sidebar-* vars also generated */
}

.dark {
  --background: oklch(0.145 0 0);
  --foreground: oklch(0.985 0 0);
  --primary: oklch(0.922 0 0);
  --primary-foreground: oklch(0.205 0 0);
  /* ... dark overrides for all vars */
}

@theme inline {
  --color-background: var(--background);
  --color-foreground: var(--foreground);
  --color-primary: var(--primary);
  --color-primary-foreground: var(--primary-foreground);
  --color-secondary: var(--secondary);
  --color-secondary-foreground: var(--secondary-foreground);
  --color-muted: var(--muted);
  --color-muted-foreground: var(--muted-foreground);
  --color-accent: var(--accent);
  --color-accent-foreground: var(--accent-foreground);
  --color-destructive: var(--destructive);
  --color-border: var(--border);
  --color-input: var(--input);
  --color-ring: var(--ring);
  --radius: var(--radius);
  /* all chart-*, sidebar-*, card-*, popover-* colours registered here */
}
```

**Naming convention:** `--primary` is background colour, `--primary-foreground` is text colour. Use as `bg-primary text-primary-foreground`.

### Adding Custom Colour Tokens

```css
:root {
  --warning: oklch(0.84 0.16 84);
  --warning-foreground: oklch(0.28 0.07 46);
}
.dark {
  --warning: oklch(0.41 0.11 46);
  --warning-foreground: oklch(0.99 0.02 95);
}
@theme inline {
  --color-warning: var(--warning);
  --color-warning-foreground: var(--warning-foreground);
}
```

Then use: `class="bg-warning text-warning-foreground"`

### Combobox (Popover + Command Pattern)

```svelte
<script lang="ts">
  import CheckIcon from "@lucide/svelte/icons/check";
  import ChevronsUpDownIcon from "@lucide/svelte/icons/chevrons-up-down";
  import { tick } from "svelte";
  import * as Command from "$lib/components/ui/command/index.js";
  import * as Popover from "$lib/components/ui/popover/index.js";
  import { Button } from "$lib/components/ui/button/index.js";
  import { cn } from "$lib/utils.js";

  const items = [
    { value: "sveltekit", label: "SvelteKit" },
    { value: "next.js", label: "Next.js" },
  ];

  let open = $state(false);
  let value = $state("");
  let triggerRef = $state<HTMLButtonElement>(null!);

  const selectedLabel = $derived(
    items.find((f) => f.value === value)?.label
  );

  function closeAndFocusTrigger() {
    open = false;
    tick().then(() => triggerRef.focus());
  }
</script>

<Popover.Root bind:open>
  <Popover.Trigger bind:ref={triggerRef}>
    {#snippet child({ props })}
      <Button {...props} variant="outline" class="w-[200px] justify-between"
        role="combobox" aria-expanded={open}>
        {selectedLabel || "Select..."}
        <ChevronsUpDownIcon class="ms-2 size-4 shrink-0 opacity-50" />
      </Button>
    {/snippet}
  </Popover.Trigger>
  <Popover.Content class="w-[200px] p-0">
    <Command.Root>
      <Command.Input placeholder="Search..." />
      <Command.List>
        <Command.Empty>No results.</Command.Empty>
        <Command.Group>
          {#each items as item}
            <Command.Item value={item.value}
              onSelect={() => { value = item.value; closeAndFocusTrigger(); }}>
              <CheckIcon class={cn("me-2 size-4", value !== item.value && "text-transparent")} />
              {item.label}
            </Command.Item>
          {/each}
        </Command.Group>
      </Command.List>
    </Command.Root>
  </Popover.Content>
</Popover.Root>
```

**Requires:** `npx shadcn-svelte@latest add popover command`

## Forms (sveltekit-superforms + Zod)

### Dependencies

```bash
npm i -D sveltekit-superforms zod
npx shadcn-svelte@latest add form input label
```

### End-to-End Pattern

**1. Schema** (`src/routes/settings/schema.ts`):
```typescript
import { z } from "zod";

export const formSchema = z.object({
  username: z.string().min(2).max(50),
  email: z.string().email(),
});

export type FormSchema = typeof formSchema;
```

**2. Server load + action** (`+page.server.ts`):
```typescript
import type { PageServerLoad, Actions } from "./$types.js";
import { superValidate } from "sveltekit-superforms";
import { zod4 } from "sveltekit-superforms/adapters";
import { formSchema } from "./schema";
import { fail } from "@sveltejs/kit";

export const load: PageServerLoad = async () => {
  return { form: await superValidate(zod4(formSchema)) };
};

export const actions: Actions = {
  default: async (event) => {
    const form = await superValidate(event, zod4(formSchema));
    if (!form.valid) return fail(400, { form });
    // process form.data
    return { form };
  },
};
```

**3. Form component** (`settings-form.svelte`):
```svelte
<script lang="ts">
  import * as Form from "$lib/components/ui/form/index.js";
  import { Input } from "$lib/components/ui/input/index.js";
  import { formSchema, type FormSchema } from "./schema";
  import {
    type SuperValidated,
    type Infer,
    superForm,
  } from "sveltekit-superforms";
  import { zod4Client } from "sveltekit-superforms/adapters";

  let { data }: { data: { form: SuperValidated<Infer<FormSchema>> } } = $props();

  const form = superForm(data.form, {
    validators: zod4Client(formSchema),
  });

  const { form: formData, enhance } = form;
</script>

<form method="POST" use:enhance>
  <Form.Field {form} name="username">
    <Form.Control>
      {#snippet children({ props })}
        <Form.Label>Username</Form.Label>
        <Input {...props} bind:value={$formData.username} />
      {/snippet}
    </Form.Control>
    <Form.Description>Your public display name.</Form.Description>
    <Form.FieldErrors />
  </Form.Field>

  <Form.Field {form} name="email">
    <Form.Control>
      {#snippet children({ props })}
        <Form.Label>Email</Form.Label>
        <Input {...props} bind:value={$formData.email} type="email" />
      {/snippet}
    </Form.Control>
    <Form.FieldErrors />
  </Form.Field>

  <Form.Button>Submit</Form.Button>
</form>
```

**4. Page** (`+page.svelte`):
```svelte
<script lang="ts">
  import SettingsForm from "./settings-form.svelte";
  let { data } = $props();
</script>

<SettingsForm {data} />
```

### Form Component Anatomy

| Component | Purpose |
|-----------|---------|
| `Form.Field` | Scopes form state for a field. Props: `{form}`, `name` |
| `Form.Control` | Renders control with correct aria attributes |
| `Form.Label` | Accessible label, linked to control |
| `Form.Description` | Helper text below input |
| `Form.FieldErrors` | Displays validation errors |
| `Form.Button` | Submit button |

**Critical:** `Form.Control` uses a `children` snippet that receives `{ props }` — spread `{...props}` on the input for proper aria bindings.

## Data Table (TanStack Table v8)

### Setup

```bash
npx shadcn-svelte@latest add table data-table
npm i @tanstack/table-core
```

### Column Definitions (`columns.ts`)

```typescript
import type { ColumnDef } from "@tanstack/table-core";
import { renderComponent, renderSnippet } from "$lib/components/ui/data-table";
import { createRawSnippet } from "svelte";

export type Payment = {
  id: string;
  amount: number;
  status: "pending" | "processing" | "success" | "failed";
  email: string;
};

export const columns: ColumnDef<Payment>[] = [
  { accessorKey: "status", header: "Status" },
  { accessorKey: "email", header: "Email" },
  {
    accessorKey: "amount",
    header: () => {
      const snippet = createRawSnippet(() => ({
        render: () => `<div class="text-end">Amount</div>`,
      }));
      return renderSnippet(snippet);
    },
    cell: ({ row }) => {
      const fmt = new Intl.NumberFormat("en-US", { style: "currency", currency: "USD" });
      const snippet = createRawSnippet<[number]>((getVal) => ({
        render: () => `<div class="text-end font-medium">${fmt.format(getVal())}</div>`,
      }));
      return renderSnippet(snippet, row.original.amount);
    },
  },
];
```

### DataTable Component (`data-table.svelte`)

```svelte
<script lang="ts" generics="TData, TValue">
  import { type ColumnDef, getCoreRowModel } from "@tanstack/table-core";
  import { createSvelteTable, FlexRender } from "$lib/components/ui/data-table";
  import * as Table from "$lib/components/ui/table";

  let { data, columns }: { columns: ColumnDef<TData, TValue>[]; data: TData[] } = $props();

  const table = createSvelteTable({
    get data() { return data; },
    columns,
    getCoreRowModel: getCoreRowModel(),
  });
</script>

<div class="rounded-md border">
  <Table.Root>
    <Table.Header>
      {#each table.getHeaderGroups() as headerGroup (headerGroup.id)}
        <Table.Row>
          {#each headerGroup.headers as header (header.id)}
            <Table.Head colspan={header.colSpan}>
              {#if !header.isPlaceholder}
                <FlexRender content={header.column.columnDef.header} context={header.getContext()} />
              {/if}
            </Table.Head>
          {/each}
        </Table.Row>
      {/each}
    </Table.Header>
    <Table.Body>
      {#each table.getRowModel().rows as row (row.id)}
        <Table.Row data-state={row.getIsSelected() && "selected"}>
          {#each row.getVisibleCells() as cell (cell.id)}
            <Table.Cell>
              <FlexRender content={cell.column.columnDef.cell} context={cell.getContext()} />
            </Table.Cell>
          {/each}
        </Table.Row>
      {:else}
        <Table.Row>
          <Table.Cell colspan={columns.length} class="h-24 text-center">No results.</Table.Cell>
        </Table.Row>
      {/each}
    </Table.Body>
  </Table.Root>
</div>
```

## Svelte 5 Patterns for shadcn-svelte

### Wrapping Components

```svelte
<!-- CustomButton.svelte -->
<script lang="ts">
  import { Button, type ButtonProps } from "$lib/components/ui/button/index.js";
  import { cn } from "$lib/utils.js";

  let { class: className, children, ...rest }: ButtonProps = $props();
</script>

<Button class={cn("custom-class", className)} {...rest}>
  {@render children?.()}
</Button>
```

### Snippets (NOT Slots)

shadcn-svelte v2 uses Svelte 5 snippets. Some components expose snippet props with parameters:

```svelte
<!-- Popover.Trigger with child snippet -->
<Popover.Trigger>
  {#snippet child({ props })}
    <Button {...props} variant="outline">Open</Button>
  {/snippet}
</Popover.Trigger>

<!-- Form.Control with children snippet -->
<Form.Control>
  {#snippet children({ props })}
    <Form.Label>Name</Form.Label>
    <Input {...props} bind:value={$formData.name} />
  {/snippet}
</Form.Control>
```

### Controlled State

Components from bits-ui support `bind:open`, `bind:value`, etc:

```svelte
<script lang="ts">
  let dialogOpen = $state(false);
  let selectValue = $state("");
</script>

<Dialog.Root bind:open={dialogOpen}>...</Dialog.Root>
<Select.Root type="single" bind:value={selectValue}>...</Select.Root>

<!-- Programmatic control -->
<Button onclick={() => { dialogOpen = true; }}>Open Dialog</Button>
```

### Passing Classes

Always use `cn()` when merging user classes with defaults:

```svelte
<script lang="ts">
  let { class: className, ...rest } = $props();
</script>
<div class={cn("default-classes", className)} {...rest} />
```

## Common Pitfalls

1. **Wrong CLI** — `npx shadcn@latest` is React. Use `npx shadcn-svelte@latest`.
2. **Vite plugin order** — `tailwindcss()` MUST come before `sveltekit()` in vite.config.ts.
3. **No tailwind.config.js** — Tailwind v4 uses CSS-first config via `@theme inline` in app.css.
4. **Import paths** — Always use `$lib/components/ui/<name>/index.js` (with `.js` extension for SvelteKit module resolution).
5. **Snippets not slots** — shadcn-svelte v2 uses Svelte 5 snippets. `{#snippet children({ props })}` not `<slot>`.
6. **Form.Control snippet** — Must spread `{...props}` from the `children` snippet onto the input element for accessibility.
7. **SSR + Dialog/Sheet open** — Don't initialise `open = $state(true)` for modals — they should start closed and open client-side to avoid hydration mismatch.
8. **Tailwind v4 content scanning** — v4 auto-detects content files. No `content: []` array needed. If classes aren't detected, ensure files are within the project root.
9. **cn() import** — Import from `$lib/utils.js` (created by init), NOT from a package.
10. **Sonner theme** — Toaster uses `mode-watcher` for dark mode. If you remove mode-watcher, also remove the `theme` prop from `<Toaster />`.
